## 无符号和有符号类型

有符号类型是指可以表示正数、负数和零的数据类型，如 byte、short、int 和 long 等。而无符号类型则只能表示非负整数，例如无符号 byte 类型可以表示范围从 0 到 255 的整数。

在 Java 中 byte、short、int 和 long 都是有符号数，它们的范围如下：

- byte：1 字节，8 位，范围为 -128 到 127
- short：2 字节，16 位，范围为 -32768 到 32767
- int：4 字节，32 位，范围为 -2147483648 到 2147483647
- long：8 字节，64 位，范围为 -9223372036854775808 到 9223372036854775807

在 C(C99 开始) 和 C++ 语言中是有无符号类型的，设备大多是采用 C 和 C++ 开发，例如：网卡端口号范围是从0到65535 就是一个无符号的 short，所以我们需要在 Java 中去兼容无符号类型：

- unsigned byte：范围为 0 到 255
- unsigned short：范围为 0 到 65535
- unsigned int：范围为 0 到 4294967295
- unsigned long：范围为 0 到 18446744073709551615

如上表可以看出，无符号类型数值大了一倍，在设计数据表主键 id 时，需要勾选上无符号。

Java 中没有直接支持无符号整数类型，但可以通过一些技巧实现对无符号数的支持。例如，使用 Java 的位运算操作来模拟对无符号整数的支持。例如，通过使用 “& 0xFF” 位运算操作符，可以将一个有符号字节值转换为无符号字节值。例如，下面的代码将一个有符号 byte 类型的值转换为无符号 int 类型的值：

```java
byte signedByte = -1;
int unsignedByte = signedByte & 0xFF; // 结果为255
```

在 Java 8 中，新增了一个无符号整数类 UnsignedLong，可用于处理 64 位无符号整数。需要注意的是，这个类并非 Java 标准 API 的一部分，而是 Optional 类库中的一个类。

## mica-net 中读取和写出无符号数据

在 mica-net（0.1.3 开始） ByteBufferUtil 工具类中添加了下列方法：
**读取：**

| 方法                 | 字节数 | 大小端          |
| -------------------- | ------ |--------------|
| readUnsignedByte     | 1 字节 | 1字节不需要考虑大小端 |
| readUnsignedShortLE  | 2 字节 | 小端           |
| readUnsignedShortBE  | 2 字节 | 大端           |
| readUnsignedMediumLE | 3 字节 | 小端           |
| readUnsignedMediumBE | 3 字节 | 大端           |
| readUnsignedIntLE    | 4 字节 | 小端           |
| readUnsignedIntBE    | 4 字节 | 大端           |
| readUnsignedLongLE   | 8 字节 | 小端           |
| readUnsignedLongBE   | 8 字节 | 大端           |

**写出：**

| 方法                 | 字节数 | 大小端          |
| -------------------- | ------ |--------------|
| writeUnsignedByte  | 1 字节 | 1字节不需要考虑大小端 |
| writeUnsignedShortLE  | 2 字节 | 小端           |
| writeUnsignedShortBE  | 2 字节 | 大端           |
| writeUnsignedMediumLE | 3 字节 | 小端           |
| writeUnsignedMediumBE | 3 字节 | 大端           |
| writeUnsignedIntLE    | 4 字节 | 小端           |
| writeUnsignedIntBE    | 4 字节 | 大端           |
| writeUnsignedLongLE   | 8 字节 | 小端           |
| writeUnsignedLongBE   | 8 字节 | 大端           |
